# 🚀 Professional WordPress Workflow — Sellio

Цей проєкт побудований на сучасній компонентній архітектурі.

---

## 🏗️ Структура Проєкту

```text
/var/www/sellio/
├── theme/                 <-- Тут твій PHP код (WP Тема)
│   ├── blocks/            <-- Кожен блок у своїй папці (Hero, Services тощо)
│   ├── acf-json/          <-- Поля ACF зберігаються тут автоматично (Git ready)
│   └── inc/               <-- Модульна логіка
├── src/                   <-- "Кухня" розробника (SCSS, JS)
├── docker/                <-- Налаштування Docker (Redis, Mailhog, WP-CLI)
└── tools/                 <-- CLI інструменти (Block Generator)
```

---

## 🛠️ Основні команди (Makefile)

Ми додали `Makefile`, щоб ти не писав довгі команди. 

### Робота з Docker:
- `make up` — Запустити проект
- `make down` — Зупинити проект
- `make logs` — Переглянути логи
- `make restart` — Перезавантажити контейнери

### Робота з WordPress:
- `make wp cmd="plugin list"` — Запустити WP-CLI команду (наприклад, перелік плагінів)
- `make wp cmd="user update admin --user_pass=new_pass"` — Змінити пароль адміна

## 🧩 Створення нового Блока (Генератор)

Замість того, щоб створювати папку, 4 файли, писати JSON та реєструвати PHP, просто напиши:

```bash
make block name=portfolio
```

Це миттєво створить папку `theme/blocks/portfolio/` з готовими файлами:
- `block.json` (Gutenberg конфіг)
- `render.php` (Шаблон)
- `style.scss` (Стилі блока)
- `script.js` (Скрипти блока)

## ⚡ Особливості Архітектури

### 1. Lazy Loading (Швидкість 100/100)
Gulp розділяє стилі кожного блока в окремі файли (`assets/css/hero.css`, `assets/css/portfolio.css`).
WordPress підключає ці файли **тільки тоді**, коли відповідний блок є на сторінці. Глобальний CSS залишається мінімальним.

### 2. ACF JSON Sync
Коли ти створюєш поля в адмінці ACF, вони автоматично зберігаються у файл `theme/acf-json/*.json`.
**Це критично для Git:** при переносі на сервер тобі не треба знову створювати поля — вони підтягнуться автоматично з файлів.

### 3. Redis & Mailhog
- **Redis:** Увімкнений для кешування об'єктів (Object Cache).
- **Mailhog:** Доступний на `http://localhost:8025`. Всі листи від WordPress (наприклад, з форм) прилітатимуть сюди.

## 🎨 Як верстати?

1. Пиши глобальні стилі (шрифти, кольори, кнопки) у `src/scss/`.
2. Пиши стилі конкретних блоків у `theme/blocks/назва/style.scss`.
3. Запусти `npm run dev`, щоб Gulp все зібрав і оновив браузер.

---

### Твоя перша дія:
Зараз напиши `make up`, зайди в адмінку (Appearance -> Themes) та активуй тему **Sellio**.
Після цього спробуй створити перший блок: `make block name=hero`.
